# 일간 뉴스 브리핑 서비스 기획서

2026년 1월

---

## 1. 서비스 개요

매일 아침, 어제 하루 동안 있었던 정치/경제/사회 뉴스를 AI가 자동으로 수집하고 요약하여 제공하는 뉴스 큐레이션 서비스

| 항목 | 내용 |
|------|------|
| 서비스명 | Daily Wrap |
| 핵심 컨셉 | "어제 하루 전체 조감도" - 스크롤 한 번으로 하루를 파악 |
| 타겟 사용자 | 뉴스를 챙겨보고 싶지만 시간이 없는 직장인 |
| 차별점 | 뉴닉 등 기존 서비스가 2-3개 주제를 깊게 다룬다면, 본 서비스는 하루 전체를 빠르게 훑을 수 있도록 구성 |

---

## 2. 수익 모델

| 구분 | 내용 |
|------|------|
| 무료 | 웹앱에서 오후에 공개 (지난 1달간 뉴스레터 열람 가능) |
| 유료 (월 2,900원) | 아침 7시 카카오톡 오픈채팅방 선발송 + 주간 정리 |

### 웹앱 기능

- 지난 일주일의 뉴스레터 아카이브 열람 가능 (무료/유료 모두)

### 베타 오픈 프로모션

- 베타 기간 (1-2개월) 동안 유료 버전 무료 제공
- 베타 테스터 100명 선착순 모집
- 베타 기간 종료 후 유료 전환 시 얼리버드 혜택 제공 예정

섹션별 중요도순 배열 방식 채택

|------|---------|------|

| 사회 | 3-5개 | 한줄 요약 + 4-5문장 설명 + 원본 링크 |

---

## 4. 기술 구현

**전체 파이프라인:** 뉴스 수집 → AI 처리 → 저장 → 웹사이트 → 오픈채팅방 발송

| 단계 | 구현 방안 |
|------|-----------|
| 뉴스 수집 | 네이버 뉴스 RSS (무료, 정치/경제/사회 카테고리별 피드) |
| AI 처리 | Anthropic Claude API (Sonnet) - 중복 제거, 중요도 판단, 요약 |
| 저장/DB | Supabase 또는 Firebase (무료 티어) |
| 웹앱 | Next.js + Vercel (무료 티어) - 1달간 뉴스레터 아카이브 제공 |
| 발송 | 카카오톡 오픈채팅방 (사업자 불필요) |

---

## 5. 저작권 대응

- 헤드라인 + AI가 재작성한 요약문 제공 (원문 문장 그대로 복붙 금지)
- 반드시 원본 링크 제공
- 뉴스 전문 게재 금지

## 8. 초기 사용자 확보 전략

1. **무료 베타 기간 운영 (1-2개월)** - "베타 테스터 100명 모집"
2. **지인 네트워크 활용** - 직장 동료, 뉴스에 관심 있는 지인 대상
3. **커뮤니티 홍보** - 블라인드, 클리앙, 트위터 등
4. **개인 블로그/SNS** - 샘플 뉴스레터 공개 및 구독 유도
